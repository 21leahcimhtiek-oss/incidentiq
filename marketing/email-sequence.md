## Email 3: Post-Mortem Education (Day 5)

**Subject:** Why 80% of post-mortems never get written (and how to fix it)

**Body:**

Hi [First Name],

Here's a pattern we see constantly:

Incident happens -> team resolves it -> everyone is exhausted -> post-mortem "gets scheduled" -> it never happens -> same root cause causes the next incident.

The problem: post-mortems are valuable but take 2-3 hours to write properly.

IncidentIQ's solution: generate them automatically.

When your incident is resolved, click "Generate Post-Mortem." Our AI reads the full incident timeline and produces:

- Executive summary
- Complete chronological timeline
- Root cause analysis
- Contributing factors
- Action items with priority
- MTTR metrics

It's not perfect — you should review and edit it. But it's 90% done in 3 seconds instead of 0% done after 3 days.

**Try it ->** Create and resolve a test incident today.
